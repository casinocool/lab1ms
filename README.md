# 1. Расчет решения (solve): O(V * P)

# 2. Построение макета (Layout): O(V * E)

# 3. Сохранение/Загрузка JSON: O(V + E)

# 4. Подсветка пути: O(D) или O(V)

# 5. Построение дерева (addNextStep): O(1) + Layout

  <ins>V (Vertices)</ins> — количество узлов в дереве.

 <ins>E (Edges)</ins> — количество ходов (ребер).

  <ins>P (Players)</ins> — количество игроков.

 <ins> D (Depth)</ins> — максимальная глубина дерева.
